What's happening (intern explanation)

The setup

Deutsche Telekom added a custom dropdown question to their Career Hub application form for internal employees applying to jobs. The question (named q_poc, labelled "Choose any") has two options: "T.E.O" and "Lidership". The config says: when an employee answers this question, write their answer into a custom field on their profile called efcustom_text_talent_pipeline_osd.

The expectation

Employee picks "T.E.O" → submits application → their profile field updates from "Executive-ready" to "T.E.O".

What actually happens

Employee picks "T.E.O" → submits application → profile field stays "Executive-ready". The answer is captured somewhere in the application data, but it never gets written back to the profile's custom field.

Why — in one sentence

The IJP submit path routes answers through a hardcoded registry that only knows about two special fields (contact_consent and skills-suggester) — it has no handler for custom_info.* fields, so the answer silently falls through and nothing is written. The external (PCS) apply flow fixed this a while ago but the fix was never ported to the internal (IJP) flow.
